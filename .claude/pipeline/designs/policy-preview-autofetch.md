# PolicyPreview Auto-Fetch Sample Inputs

## Overview

Enable the PolicyPreview API to automatically fetch sample audit logs and/or events based on the policy's resource type, eliminating the need for clients to manually query and provide inputs. This improves the developer experience for policy testing by reducing the number of API calls from 2 to 1.

## Requirements

### Functional Requirements

- FR1: PolicyPreview API accepts an optional `autoFetch` specification instead of requiring manual `inputs`
- FR2: When `autoFetch` is provided, the API queries recent audit logs and/or events matching the policy's resource type
- FR3: Fetched samples are returned in the response so clients can see what was tested
- FR4: Auto-fetch supports both audit logs and events based on policy rules present
- FR5: Auto-fetch respects configurable time range and limit parameters
- FR6: Manual `inputs` continue to work for advanced testing scenarios
- FR7: At least one of `inputs` or `autoFetch` must be provided

### Non-Functional Requirements

- NFR1: Auto-fetch queries complete within 2 seconds for typical datasets
- NFR2: Auto-fetch does not impact manual input performance
- NFR3: Error messages clearly distinguish between no matching data vs. query failures
- NFR4: Implementation reuses existing ClickHouse storage infrastructure

## Design

No new resources. This feature extends the existing `PolicyPreview` ephemeral resource.

```go
type AutoFetchSpec struct {
	// Limit is the maximum number of sample inputs to fetch (default: 10, max: 50).
	// The API fetches up to this many audit logs and/or events.
```

### Storage Design

**Auto-Fetch Implementation** (`internal/registry/activity/preview/storage.go`):

The `PolicyPreviewStorage.Create()` method will be updated to:

1. **Validate spec**: Check that exactly one of `inputs` or `autoFetch` is provided
2. **Handle auto-fetch**: If `autoFetch` is specified:
   - Determine which sources to query based on `sources` field and policy rules
   - Query ClickHouse for matching samples
   - Convert results to `PolicyPreviewInput` format
   - Store in status.fetchedInputs
3. **Evaluate policy**: Use fetched or manual inputs for evaluation
4. **Return result**: Include fetched inputs in status if applicable

**ClickHouse Integration**:

Auto-fetch will use the existing `ClickHouseStorage` interface:

**Auto-Fetch Query Logic**:

```go
// autoFetchInputs retrieves sample inputs based on the policy resource type.
func (s *PolicyPreviewStorage) autoFetchInputs(
	ctx context.Context,
	spec *v1alpha1.PolicyPreviewSpec,
	scope storage.ScopeContext,
) ([]v1alpha1.PolicyPreviewInput, error) {
	inputs := []v1alpha1.PolicyPreviewInput{}
	autoFetch := spec.AutoFetch

	// Determine time range
	timeRange := autoFetch.TimeRange
	if timeRange == "" {
		timeRange = "24h"
	}
	startTime := fmt.Sprintf("now-%s", timeRange)
	endTime := "now"

	// Determine limit
	limit := autoFetch.Limit
	if limit == 0 {
		limit = 10
	}

	// Determine sources
	sources := autoFetch.Sources
	if sources == "" {
		sources = "both"
	}

	// Fetch audit logs if requested and policy has audit rules
	if (sources == "audit" || sources == "both") && len(spec.Policy.AuditRules) > 0 {
		auditInputs, err := s.fetchAuditLogSamples(ctx, spec, startTime, endTime, limit, scope)
		if err != nil {
			return nil, fmt.Errorf("failed to fetch audit log samples: %w", err)
		}
		inputs = append(inputs, auditInputs...)
	}

	// Fetch events if requested and policy has event rules
	if (sources == "events" || sources == "both") && len(spec.Policy.EventRules) > 0 {
		eventInputs, err := s.fetchEventSamples(ctx, spec, startTime, endTime, limit, scope)
		if err != nil {
			return nil, fmt.Errorf("failed to fetch event samples: %w", err)
		}
		inputs = append(inputs, eventInputs...)
	}

	return inputs, nil
}

// fetchAuditLogSamples queries ClickHouse for audit logs matching the policy resource.
func (s *PolicyPreviewStorage) fetchAuditLogSamples(
	ctx context.Context,
	spec *v1alpha1.PolicyPreviewSpec,
	startTime, endTime string,
	limit int32,
	scope storage.ScopeContext,
) ([]v1alpha1.PolicyPreviewInput, error) {
	resource := spec.Policy.Resource

	// Build filter to match policy resource
	filter := fmt.Sprintf(
		`objectRef.apiGroup == "%s" && objectRef.resource contains "%s"`,
		resource.APIGroup,
		strings.ToLower(resource.Kind),
	)

	// Query audit logs
	querySpec := v1alpha1.AuditLogQuerySpec{
		StartTime: startTime,
		EndTime:   endTime,
		Filter:    filter,
		Limit:     limit,
	}

	result, err := s.clickhouse.QueryAuditLogs(ctx, querySpec, scope)
	if err != nil {
		return nil, err
	}

	// Convert to PolicyPreviewInput
	inputs := make([]v1alpha1.PolicyPreviewInput, len(result.Events))
	for i, event := range result.Events {
		inputs[i] = v1alpha1.PolicyPreviewInput{
			Type:  "audit",
			Audit: &event,
		}
	}

	return inputs, nil
}

// fetchEventSamples queries ClickHouse for K8s events matching the policy resource.
func (s *PolicyPreviewStorage) fetchEventSamples(
	ctx context.Context,
	spec *v1alpha1.PolicyPreviewSpec,
	startTime, endTime string,
	limit int32,
	scope storage.ScopeContext,
) ([]v1alpha1.PolicyPreviewInput, error) {
	// Build field selector to match policy resource
	resource := spec.Policy.Resource
	fieldSelector := fmt.Sprintf(
		"regarding.apiVersion=%s/v1,regarding.kind=%s",
		resource.APIGroup,
		resource.Kind,
	)

	// Query events
	querySpec := storage.EventQuerySpec{
		StartTime:     startTime,
		EndTime:       endTime,
		FieldSelector: fieldSelector,
		Limit:         limit,
	}

	result, err := s.clickhouse.QueryEvents(ctx, querySpec, scope)
	if err != nil {
		return nil, err
	}

	// Convert to PolicyPreviewInput
	inputs := make([]v1alpha1.PolicyPreviewInput, len(result.Events))
	for i, event := range result.Events {
		eventJSON, _ := json.Marshal(event)
		inputs[i] = v1alpha1.PolicyPreviewInput{
			Type:  "event",
			Event: &runtime.RawExtension{Raw: eventJSON},
		}
	}

	return inputs, nil
}
```

### Validation Changes

**Updated Validation** (`internal/registry/activity/preview/storage.go`):

```go
// validatePolicyPreview validates the entire PolicyPreview spec.
func validatePolicyPreview(preview *v1alpha1.PolicyPreview) field.ErrorList {
	allErrs := field.ErrorList{}
	specPath := field.NewPath("spec")

	// Validate policy spec (existing logic)
	internalSpec := convertPolicySpecToInternal(&preview.Spec.Policy)
	allErrs = append(allErrs, policy.ValidateActivityPolicySpec(internalSpec, specPath.Child("policy"))...)

	// Validate that exactly one of inputs or autoFetch is provided
	hasInputs := len(preview.Spec.Inputs) > 0
	hasAutoFetch := preview.Spec.AutoFetch != nil

	if !hasInputs && !hasAutoFetch {
		allErrs = append(allErrs, field.Required(specPath, "provide either 'inputs' or 'autoFetch'"))
	}

	if hasInputs && hasAutoFetch {
		allErrs = append(allErrs, field.Invalid(
			specPath,
			preview.Spec,
			"cannot specify both 'inputs' and 'autoFetch' - use one or the other",
		))
	}

	// Validate inputs if provided (existing logic)
	if hasInputs {
		allErrs = append(allErrs, validatePreviewInputs(preview.Spec.Inputs, specPath.Child("inputs"))...)
	}

	// Validate autoFetch if provided
	if hasAutoFetch {
		allErrs = append(allErrs, validateAutoFetch(preview.Spec.AutoFetch, specPath.Child("autoFetch"))...)
	}

	return allErrs
}

// validateAutoFetch validates the AutoFetch spec.
func validateAutoFetch(autoFetch *v1alpha1.AutoFetchSpec, fldPath *field.Path) field.ErrorList {
	allErrs := field.ErrorList{}

	// Validate limit
	if autoFetch.Limit < 0 {
		allErrs = append(allErrs, field.Invalid(fldPath.Child("limit"), autoFetch.Limit, "must be >= 0"))
	}
	if autoFetch.Limit > 50 {
		allErrs = append(allErrs, field.Invalid(fldPath.Child("limit"), autoFetch.Limit, "must be <= 50"))
	}

	// Validate timeRange
	if autoFetch.TimeRange != "" {
		if _, err := timeutil.ParseRelativeTime(autoFetch.TimeRange); err != nil {
			allErrs = append(allErrs, field.Invalid(
				fldPath.Child("timeRange"),
				autoFetch.TimeRange,
				"must be a valid relative time (e.g., '1h', '24h', '7d')",
			))
		}
	}

	// Validate sources
	if autoFetch.Sources != "" {
		validSources := []string{"audit", "events", "both"}
		if !contains(validSources, autoFetch.Sources) {
			allErrs = append(allErrs, field.NotSupported(
				fldPath.Child("sources"),
				autoFetch.Sources,
				validSources,
			))
		}
	}

	return allErrs
}
```

## Handoff

### Decisions Made

**Decision 1: Auto-fetch supports both audit and events**
- Rationale: Policies often have both audit and event rules. Supporting both provides maximum flexibility.
- Implementation: `sources` field defaults to "both" but allows filtering to specific types.

**Decision 2: Fetched inputs are returned in status**
- Rationale: Clients need to see what data was tested for debugging and transparency.
- Implementation: `fetchedInputs` array in `PolicyPreviewStatus` contains the auto-fetched samples.

**Decision 3: Limit auto-fetch to 50 samples maximum**
- Rationale: Prevents excessive query load and ensures reasonable response times.
- Implementation: Validation enforces `limit <= 50`, default is 10.

**Decision 4: Default time range is 24 hours**
- Rationale: Balance between finding recent data and keeping query scope reasonable.
- Implementation: `timeRange` defaults to "24h" if not specified.

**Decision 5: Empty results are not errors**
- Rationale: It's valid for a new policy to have no matching historical data.
- Implementation: Return empty `fetchedInputs` array with success status, not an error.

**Decision 6: Inputs and autoFetch are mutually exclusive**
- Rationale: Simplifies API semantics and prevents confusion about which data is tested.
- Implementation: Validation ensures exactly one is provided.

**Decision 7: Reuse existing ClickHouse storage interface**
- Rationale: Avoids duplicating query logic and ensures consistent behavior.
- Implementation: Inject `ClickHouseStorage` into `PolicyPreviewStorage`.

### Open Questions

**Question 1: Should we cache auto-fetched samples?**
- Status: Non-blocking (can proceed with no caching)
- Context: Repeated previews might fetch the same data multiple times
- Recommendation: Start without caching, add if performance issues arise

**Question 2: Should we support custom filters for auto-fetch?**
- Status: Non-blocking (can proceed without custom filters)
- Context: Advanced users might want to filter auto-fetched data beyond resource type
- Recommendation: Add in future iteration if requested
